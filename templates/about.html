About Name Generator

Welcome to Name Generator, my final project for CS50x!

Name Generator is a program that uses a Markov chain open_in_new to generate random names.  Markov chains are essentially a way to choose a random next state based on the previous state.  However, instead of simply choosing random states, a Markov chain takes a large set of training data as input.  It processes this input, and then chooses each next state based on the probabilities found in the training data.  As it relates to Name Generator, the input is a corpus of thousands of names in the selected language.  Each letter in the name is a state, and as the program generates the name, each new letter is chosen based on the previous letter.

Here's an example using three common English words: bat, bay, and bug.  To process these names, you simply write down each letter that occurs in the corpus, along with each letter which follows that letter.  In this case, you get:

b | a, a, u
a | t, y
t | .
y | .
u | g
g | .

Notice that some possibilities are not letters.  The period character (.) is the end state, which marks the end of the word.  Before processing the word, I add the end state to the end of each name.  Once the program chooses the end state, the word ends, and no more letters are chosen.

Now that we have created this table of probabilities, we can start generating names.  We start by choosing a totally random starting state.  For example, we may choose the letter b.  You can see that the b state has three possible states following it: a, a, and u.  This is because, in the input text (bat, bay, bug), the letter b appears three times, with those three letters following it in each case.  Now, we randomly choose the next state.  a will be twice as likely to be chosen as u.  If a is chosen next, we next have two possibilities: t and y.  If t is chosen, we now only have one choice, the end state.  At this point, we have generated a totally random word, bay from the input text, bat, bay, bug.

In this case, the word is identical to one of the input words.  That is always a possibility, but it becomes much less likely once we start using a larger input text.  The inputs that my program uses are between 600 and 5,000 names, depending on how many I could find.  For example, using the German input, which has 2,281 names in it, we get much more varied responses.  For example, my program outputted five German names: Schöningen, Geln, Bauterbrüc, Werden, and Nienhein.  Only one of those, Schöningen, already existed in the input.
